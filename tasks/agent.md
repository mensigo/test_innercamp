# Постановка задачи

## Общее

Пишем простого агента по теме обучения в вузе (студенты, предметы, успеваемость и тд).  
Можно сказать, что студенты уже прошли экзамены и находятся на каникулах.

Агент должен уметь отвечать на некоторые типовые вопросы. Проверка осуществляется через набор end-to-end тестов.

## Глобальная цель

Написать функцию agent, которая будет принимать на вход запрос пользователя и возвращать ответ агента (строка + экстра инфо). Подробнее см. в секции "Что нужно реализовать".

## Что дано

- данные
- апи к ним
- тесты

Предполагается, что агент сможет извлекать интент/параметры из запроса пользователя и запрашивать необходимые данные через существующие апи.

### API

Представлены в виде python функций в директории src/api.

#### 1. get_top_students

- на вход название предмета и топ-k (дефолт k=3)
- возвращает информацию топ-k студентов, отсортированных по баллу (при равном балле - по имени)
- пример запроса: `get_top_students("Machine Learning")`
- пример ответа: `[ {"name": "Васильев Артем", "score": 4.9}, {"name": "Павлова Ирина", "score": 4.8}, {"name": "Новикова Дарья", "score": 4.7} ]`
- !! если название предмета пусто / нет в данных, то возвращает пустой список
- !! если значение k < 1, то возвращает пустой список
- !! если значение k > 10, то падает с ошибкой

#### 2. get_avg_score

- на вход название предмета
- возвращает средний балл студентов по указанному предмету
- пример запроса: `get_avg_score("Probability Theory")`
- пример ответа: `{ "avg_score": 4.1 }`
- !! если название предмета пусто / нет в данных, то возвращает пустой словарь

#### 3. get_avg_overall_score

- на вход ничего
- возвращает средний балл студентов по всем предметам
- пример запроса: `get_avg_overall_score()`
- пример ответа: `{ "avg_score": 4.0 }`

#### 4. vector_search

- на вход строка-запрос и топ-k (дефолт k=2)
- возвращает набор чанков
- пример запроса: `vector_search("лектор по мл")`
- пример ответа: `{ "chunks":  [ "text1", "text2" ] }`

## Что нужно реализовать

### Агент

В директории src есть agent.py с функцией agent - ее нужно написать. В ней зашита вся механика с вызовом API, принятием решений и тд.

Пример запроса пользователя:

```python
user_request = "топ студентов по мл"
```

Пример ответа функции agent:

```python
agent_result = {
    "answer": "Васильев Артем (4.9), Павлова Ирина (4.8), Новикова Дарья (4.7)",
    "token_usage": {
        "prompt_tokens": 18,
        "completion_tokens": 24,
        "total_tokens": 42
    }
}
```

Добавление token_usage пока опционально.

### Эмбеддинги

Работа API vector_search завязана на индексе, для построения которого нужны эмбеддинги.

В директории src есть utils.py с функцией get_embeddings - ее нужно реализовать. Затем запустить src/prepare_data.py для построения индекса.

Пример запроса для `get_embeddings`:

```python
{'input': 'машинное обучение'}
```

Пример ответа (для размерности эмбеддингов d=2):

```json
[0.935546875, -0.092529296]
```

# Начало работы (пример)

1. Выберите среду выполнения:
  - Локально
  - В песочнице (см. [sigma_sandbox.md](sigma_sandbox.md) для подробностей)
2. Установите Python (желательно через sberusersoft, рекомендованный - 3.11).
3. Установите зависимости из `pyproject.toml`:
  - Можно использовать виртуальное окружение или глобальную установку.
  - Настройте index-url для sberosc (см. [sigma_sberosc.md](sigma_sberosc.md)).
  - Установите пакеты через `pip install .`
  - Опционально: можно через `uv sync`, если uv настроен.
4. Подготовьте индекс для работы с markdown-документами:
  - Реализуйте функцию `get_embeddings` в `src/utils.py`.
  - Проверьте реализацию через `pytest -m llm` (все pytest метки есть в pyproject.toml).
  - Постройте индекс командой: `python src/prepare_data.py`.
5. Запустите тесты:
  - Проверьте базовые функции командой: `pytest -m "api or unit"` (эти тесты должны пройти успешно).
  - Проверьте функцию agent командой: `pytest -m agent` (эти тесты должны падать, т.к. функция agent() еще не реализована).
  - Опционально: можно запускать отдельные тестовые файлы или выбирать нужные тесты с помощью параметра `-k`. Например, команда `pytest -k search` выполнит только те тесты, в названии которых есть подстрока `search`.

Теперь можно начинать добавлять логику агента в src/agent.py, ориентируясь на кейсы из тестов test/agent.
**Критерий** прохождения задания: все тесты из директории test/ проходят успешно (запуск `pytest`).
